# 系统问答与操作手册管理平台

## 项目概述

这是一个纯前端实现的系统问答与操作手册管理平台，提供管理员和用户两种角色的界面。系统通过浏览器本地存储（LocalStorage）实现数据持久化，无需后端服务即可独立运行。

### 技术栈

- **前端框架**：原生 JavaScript（无框架依赖）
- **样式**：CSS3（Flexbox 布局）
- **数据存储**：LocalStorage
- **外部库**：
  - [SheetJS (xlsx)](https://cdn.jsdelivr.net/npm/xlsx@0.18.5) - Excel 导出
  - [JSZip](https://cdn.jsdelivr.net/npm/jszip@3.10.1) - ZIP 压缩包生成
  - [html-docx-js](https://cdn.jsdelivr.net/npm/html-docx-js@0.3.1) - Word 文档生成

### 文件结构

```
├── admin.html          # 管理员界面入口
├── admin.js            # 管理员端功能逻辑
├── user.html           # 用户界面入口
├── user.js             # 用户端功能逻辑
├── data.js             # 数据存储与共享模块
├── styles.css          # 全局样式
├── 系统功能菜单.txt     # 示例菜单文件
└── README.md           # 项目文档
```

## 核心功能详解

### 一、数据存储模块（data.js）

**开发逻辑**：采用 LocalStorage 实现数据的持久化存储，通过 JSON 序列化和反序列化完成数据的读写。

**核心数据结构**：

| 存储 Key                  | 说明              |
| ----------------------- | --------------- |
| `manual_menu_tree`      | 菜单树结构           |
| `manual_qa_content`     | 问答内容（按节点ID索引）   |
| `manual_manual_content` | 操作手册内容（按节点ID索引） |
| `manual_votes`          | 投票数据            |
| `manual_node_config`    | 节点配置（可见性、URL等）  |

**关键函数**：

| 函数                                    | 功能              |
| ------------------------------------- | --------------- |
| `getMenuTree()` / `saveMenuTree()`    | 获取/保存菜单树        |
| `findNodeById()` / `findParentNode()` | 树形结构查找          |
| `getNodeLevel()`                      | 计算节点层级深度        |
| `isUnderFuncRoot()`                   | 判断节点是否在功能菜单根目录下 |
| `parseMenuFile()`                     | 解析 TXT 菜单文件     |
| `buildMenuTree()`                     | 构建菜单树结构         |
| `mergeMenuTrees()`                    | 增量合并菜单（核心算法）    |
| `cascadeVisibility()`                 | 级联设置节点可见性       |
| `regenerateNodeIds()`                 | 节点ID重生成（用于失效目录） |

### 二、管理员端功能（admin.js）

#### 1. 登录认证

**开发逻辑**：硬编码账号密码验证，登录状态存储在 localStorage 实现会话保持。

- **账号**：`admin`
- **密码**：`admin123`
- **会话保持**：登录成功后将状态写入 `localStorage.adminLoggedIn`

#### 2. 菜单管理

**上传菜单文件**：`handleMenuFileUpload()`

- 仅支持 `.txt` 格式文件
- 文件格式：管道符分隔的表格（ID | 名称 | 父ID | 层级）
- 解析函数：`parseMenuFile()` 逐行处理，跳过前两行表头

**增量更新算法**：`mergeMenuTrees()`

- 遍历新菜单，识别已有节点和新增节点
- 已存在节点保留现有配置和内容
- 新增节点添加到对应父节点下
- 已删除节点（如有内容）移动到"失效菜单根目录"，同时重生成节点ID并迁移关联内容
- 无内容的已删除节点直接移除

**自定义节点**：

- 根目录类型：`custom`（自定义）、`func`（功能菜单）、`invalid`（失效）
- 支持在"自定义根目录"和"失效菜单根目录"下创建子节点
- 层级限制：最多 3 级子目录（即根 + 3 级）
- 节点创建：`showAddNodeModal()` → `addNode()`

**节点重命名**：双击节点名称触发 `editNodeName()`

**节点删除**：`deleteNode()` → `cascadeDelete()`

- 检查节点及子节点是否有内容
- 有内容弹出二次确认
- 无内容直接删除
- 删除时同步清理关联的问答、操作手册、投票、配置数据

#### 3. 拖拽功能

**目录树拖拽**：`initDragAndDrop()` + `handleDrop()`

拖拽行为规则：

| 拖拽位置          | 行为         |
| ------------- | ---------- |
| 节点上半部（<33%）   | 插入到目标节点之前  |
| 节点下半部（>67%）   | 插入到目标节点之后  |
| 节点中间（33%-67%） | 作为目标节点的子节点 |

**限制规则**：

- 功能菜单根目录节点：**只能同父节点内排序**，不能改变父子关系
- 拖拽到功能菜单节点：限制失败，显示错误提示
- 最大层级限制：根目录 - 一级 - 二级 - 三级（4级），第三级节点不能添加子节点
- 拖拽至第三级以下：限制失败，显示错误提示

**内容项拖拽**：在内容列表内拖拽调整顺序，逻辑同上

#### 4. 节点配置

**配置面板**：`renderNodeConfig()`

| 配置项     | 说明              |
| ------- | --------------- |
| 节点名称    | 可修改，功能菜单根节点不可修改 |
| 用户可见    | 关闭后子节点也将不可见     |
| 游客可见    | 需先开启"用户可见"      |
| 节点关联URL | 用于用户端URL自动定位功能  |

**保存处理**：`saveNodeConfigHandler()`

- 更新节点名称和配置
- 如关闭可见性，调用 `cascadeVisibility()` 级联关闭子节点

#### 5. 内容管理

**富文本编辑器**：`showContentModal()`

- 使用 `contenteditable` div 实现
- 工具栏支持：`bold`、`italic`、`underline`、`insertImage`
- 图片以 Base64 格式插入

**内容类型**：

- **问答（QA）**：常见问题解答
- **操作手册**：详细操作步骤

**内容数据结构**：

```javascript
{
  id: "qa_1234567890",
  title: "内容标题",
  body: "<p>HTML内容</p>",
  updateTime: "2024/1/1 12:00:00",
  updater: "管理员",
  attachment: { name: "文件.pdf" } // 仅演示，不实际存储
}
```

**更换节点功能**：

- 进入选择模式：勾选要移动的内容项
- 弹出节点选择器：支持搜索、树形展开
- 确认后批量移动内容到目标节点

**内容编辑方式**：

- 点击内容项任意位置直接打开编辑弹窗（删除按钮、拖拽手柄、统计数字除外）

#### 6. 内容指示器

**开发逻辑**：在目录树节点上显示圆点标记

| 指示器   | 颜色 | 条件             |
| ----- | -- | -------------- |
| 直接内容  | 绿色 | 当前节点有问答或操作手册内容 |
| 子节点内容 | 蓝色 | 子节点（任意层级）有内容   |

#### 7. 批量导出

**导出Q\&A**：`downloadQA()`

- 遍历整个菜单树
- 收集所有问答内容，拼接完整路径（根目录→一级→二级→三级）
- 使用 SheetJS 生成 Excel 文件
- 文件名：`系统Q&A导出_YYYYMMDDHHmmss.xlsx`

**导出操作手册**：`downloadManual()`

- 遍历菜单树收集所有操作手册内容
- 生成 ZIP 压缩包，包含：
  - `操作手册汇总.xlsx` - 汇总表格
  - `{路径}-{标题}.docx` - 每个内容项一个 Word 文件
- 使用 html-docx-js 将 HTML 转为 Word

#### 8. 投票统计

- 在内容列表显示每条内容的点赞/点踩数量
- 点击数字弹出明细弹窗 `showVoteDetail()`

### 三、用户端功能（user.js）

#### 1. 登录认证

- **账号**：`user`
- **密码**：`user123`
- **游客模式**：未登录用户可访问"游客可见"的节点内容

#### 2. 目录导航

**可见性过滤**：`renderUserTreeNode()`

- 仅显示 `config.visible = true` 的节点
- 未登录用户只能看 `config.guestVisible = true` 的节点

**URL自动定位**：`autoLocateNode()`

- 获取 URL 参数 `from` 或 `document.referrer`
- 在节点配置中查找匹配的 URL
- 自动展开并选中对应节点

**节点展开**：

- 点击整行触发展开/折叠
- 展开时自动折叠所有子节点的展开状态

#### 3. 内容展示

**折叠面板（Accordion）**：

- 每个内容项为可折叠卡片
- 点击标题展开/折叠
- 一次只展开一个内容项

**动态标签页**：

- 只显示有内容的标签（Q\&A / 操作手册）
- 当前内容为空时自动切换到另一标签

#### 4. 反馈功能

**点赞**：`handleVote()`

- 点击点赞按钮切换状态
- 已点赞再点击取消
- 点赞时自动取消点踩

**点踩**：`handleVote()` → `submitDislike()`

- 点击点踩按钮显示反馈输入框
- 填写反馈后提交
- 点踩可附带意见反馈

{

## 注意事项

1. **数据持久化**：所有数据存储在浏览器 LocalStorage，清除浏览器数据会导致数据丢失
2. **浏览器兼容性**：建议使用现代浏览器（Chrome、Firefox、Edge、Safari）
3. **附件功能**：仅做演示，实际不存储文件
4. **投票限制**：基于本地存储的固定用户ID，清除数据会重置投票状态
5. **功能限制**：
   - 最多 4 级目录结构（根 + 3 级子目录）
   - 功能菜单节点不能改变父子关系
   - 拖拽操作仅支持同一父节点内的重排序
6. **导出依赖**：导出功能依赖外部 CDN 库，需保持网络连接
