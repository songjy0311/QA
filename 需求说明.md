# 系统问答与操作手册管理平台 - 需求说明

## 1. 项目简介

### 1.1 项目背景

企业内部存在大量系统操作手册和常见问答内容，管理员需要统一管理和维护这些内容，普通用户需要便捷地查阅和使用这些资料。本项目旨在提供一个系统问答与操作手册管理平台，支持多级目录管理、内容编辑、批量导出和用户反馈功能。

### 1.2 项目目标

- 提供管理员端和用户端两种角色界面
- 实现菜单结构的灵活管理与内容的便捷维护
- 支持批量导出功能，便于离线使用和分享
- 提供用户反馈机制，持续优化内容质量

### 1.3 术语定义

| 术语 | 定义 |
|------|------|
| 菜单树 | 系统的层级目录结构，用于组织问答和操作手册 |
| 问答（Q&A） | 常见问题的解答内容 |
| 操作手册 | 系统功能的详细操作步骤文档 |
| 内容指示器 | 目录树节点上的圆点标记，表示该节点或子节点是否有内容 |

### 2.1 角色说明

| 角色 | 说明 | 默认账号 |

| 管理员 | 负责菜单结构管理、内容编辑维护、导出数据 | admin / admin123 |

| 查看问答内容 | ✓ | ✓ | ✓ |

| 编辑内容 | ✓ | ✗ | ✗ |

## 3. 功能需求

### 3.1 管理员端功能需求

#### 3.1.1 登录认证

- **功能描述**：管理员需登录后才能访问管理功能
- **具体要求**：
  - 账号密码验证
  - 登录状态持久化（关闭浏览器后重新打开仍保持登录）
  - 支持退出登录功能
  - 登录界面显示当前登录用户名

#### 3.1.2 菜单文件上传

- **功能描述**：支持批量导入菜单结构
- **具体要求**：
  - 仅支持 `.txt` 格式文件
  - 文件格式：管道符分隔的表格（ID | 名称 | 父ID | 层级）
  - 第一行为表格标题，第二行为分隔线，从第三行开始为数据
  - 上传后自动解析并更新菜单结构

#### 3.1.3 增量更新

- **功能描述**：菜单文件上传时执行增量更新，而非全量替换
- **具体要求**：
  - 已有节点保留现有内容和配置
  - 新增节点追加到对应位置
  - 从菜单文件中移除的节点：
    - 如有关联内容，自动移动至"失效菜单根目录"
    - 节点ID需重新生成，内容和配置随之迁移
    - 无内容则直接删除
  - 更新完成后显示处理结果提示

#### 3.1.4 自定义目录管理

- **功能描述**：支持管理员创建自定义目录结构
- **具体要求**：
  - 系统预置三个根目录：
    - 自定义根目录（可添加子节点）
    - 功能菜单根目录（由文件上传生成）
    - 失效菜单根目录（存储已过时的节点）
  - 根目录按固定顺序排列：自定义 > 功能菜单 > 失效菜单
  - 支持在自定义根目录和失效菜单根目录下创建子节点
  - 支持创建多级子目录

#### 3.1.5 目录层级限制

- **功能描述**：限制目录的层级深度
- **具体要求**：
  - 最多支持4级目录结构（根目录 + 3级子节点）
  - 功能菜单节点默认可创建无限层级，但受拖拽规则限制

#### 3.1.6 节点操作

| 操作 | 要求 |
|------|------|
| 查看节点 | 点击目录树节点，在配置面板显示节点详情 |
| 新增子节点 | 仅在自定义根目录和失效菜单根目录下可用，层级<3级 |
| 重命名节点 | 双击节点名称进入编辑状态 |
| 删除节点 | 删除时检查是否有内容，有内容需二次确认，删除操作不可逆 |

#### 3.1.7 节点拖拽

- **功能描述**：支持通过拖拽调整目录结构
- **具体要求**：

**拖拽位置判定**：
| 拖拽位置 | 行为 |
|---------|------|
| 节点上半部（<33%） | 插入到目标节点之前 |
| 节点下半部（>67%） | 插入到目标节点之后 |
| 节点中间（33%-67%） | 作为目标节点的子节点 |

**限制规则**：
- 功能菜单根目录节点只能同父节点内排序，不能改变父子关系
- 最大层级限制为4级（根+3级），超出层级限制的拖拽操作需提示错误
- 拖拽操作需提供视觉反馈（拖拽元素高亮、目标位置指示）

#### 3.1.8 节点配置

- **功能描述**：管理单个节点的属性和行为
- **具体要求**：

| 配置项 | 说明 |
|-------|------|
| 节点名称 | 显示/修改节点名称，功能菜单根节点不可修改 |
| 用户可见 | 控制节点是否对登录用户可见，默认关闭 |
| 游客可见 | 控制节点是否对游客可见，需先开启"用户可见" |
| 节点关联URL | 输入来源系统URL，用于用户端自动定位功能 |

- **级联设置**：关闭"用户可见"时，自动关闭所有子节点的可见性

#### 3.1.9 内容管理

**内容类型**：
| 类型 | 说明 |
|------|------|
| 问答（Q&A） | 常见问题的解答 |
| 操作手册 | 功能操作的详细步骤 |

**内容编辑**：
- 支持富文本编辑（加粗、斜体、下划线、图片插入）
- 内容标题最多100字符
- 支持上传附件（仅演示用途）

**内容列表操作**：
| 操作 | 要求 |
|------|------|
| 新增内容 | 选择节点后，点击新增按钮 |
| 编辑内容 | 点击内容项任意位置（删除按钮、拖拽手柄、统计数字除外）直接打开编辑框 |
| 删除内容 | 删除需二次确认，不可恢复 |
| 拖拽排序 | 同节点内通过拖拽调整顺序 |
| 更换节点 | 勾选内容后，选择目标节点批量移动 |

#### 3.1.10 内容指示器

- **功能描述**：在目录树节点上显示内容标记
- **具体要求**：

| 指示器 | 样式 | 显示条件 |
|-------|------|---------|
| 直接内容 | 绿色圆点 | 当前节点有问答或操作手册内容 |
| 子节点内容 | 蓝色圆点 | 子节点（任意层级）有内容 |

#### 3.1.11 批量导出

**导出问答**：
- 格式：Excel（.xlsx）
- 内容包含：目录路径、内容标题、内容正文、更新时间、更新人、点赞数、点踩数
- 文件名格式：`系统Q&A导出_YYYYMMDDHHmmss.xlsx`

**导出操作手册**：
- 格式：ZIP压缩包
- 包含内容：
  - `操作手册汇总.xlsx` - 所有手册的汇总表格
  - `{目录路径}-{标题}.docx` - 每个内容项一个Word文档
- 文件名格式：`操作手册导出_YYYYMMDDHHmmss.zip`

#### 3.1.12 投票统计

- **功能描述**：查看内容的用户反馈数据
- **具体要求**：
  - 在内容列表显示每条内容的点赞/点踩数量
  - 点击数量可查看明细（点赞/点踩用户列表）
  - 点踩明细需显示用户提交的反馈意见

### 3.2 用户端功能需求

#### 3.2.1 登录认证

- **功能描述**：用户可登录获取完整功能
- **具体要求**：
  - 登录用户可访问所有"用户可见"的节点内容
  - 支持游客模式：未登录用户可访问标记为"游客可见"的内容
  - 登录状态持久化
  - 支持取消登录

#### 3.2.2 目录导航

- **功能描述**：用户浏览和定位内容
- **具体要求**：
  - 仅显示当前用户有权限访问的节点
  - 点击节点整行触发展开/折叠
  - 展开节点时自动折叠所有子节点
  - 显示内容指示器

#### 3.2.3 URL自动定位

- **功能描述**：通过URL参数自动定位到对应节点
- **具体要求**：
  - 支持通过 `from` 参数或 `referrer` 匹配节点配置的URL
  - 定位成功后自动展开目录树并选中目标节点
  - 用于与外部系统集成

#### 3.2.4 内容展示

- **功能描述**：展示问答和操作手册内容
- **具体要求**：
  - 采用折叠面板（Accordion）形式展示内容列表
  - 点击标题展开/折叠内容详情
  - 一次只展开一个内容项
  - 内容默认折叠，保持展开状态

**标签页**：
- 支持问答和操作手册两个标签页
- 只显示有内容的标签
- 当前内容为空时自动切换到另一标签

#### 3.2.5 反馈功能

**点赞**：
- 用户可对内容进行点赞/取消点赞
- 点赞时自动取消该内容的点踩状态
- 点赞状态需持久化保存

**点踩**：
- 用户可对内容进行点踩
- 点踩时弹出反馈意见输入框
- 提交反馈时需填写意见内容
- 点踩状态持久化保存

## 4. 技术架构

### 4.1 前端技术栈

- **框架**：Vue 3 + TypeScript
- **UI组件**：Element Plus
- **状态管理**：Pinia
- **富文本编辑器**：Quill
- **构建工具**：Vite

### 4.2 后端技术栈

- **框架**：Spring Boot 3.2
- **语言**：Java 21
- **ORM**：MyBatis-Plus
- **数据库**：MySQL 8.0
- **安全**：Spring Security + JWT
